# pacs008-loader-mt103 v0.0.2

Maintenance release keeping the loader in lockstep with the
[`pacs008`](https://github.com/sebastienrousseau/pacs008) core library.

## Changed

- Requires `pacs008 >= 0.0.7, < 1` (was `>= 0.0.5`) to pick up the
  0.0.7 validation bug fix. No API or behaviour changes in this loader.

## Public API (unchanged)

```python
from pacs008_loader_mt103 import parse_mt103

records = parse_mt103(mt103_text)   # -> one-element list[dict]
```

Each record's keys are exactly the flat-record fields the `pacs008`
library validates against the `pacs.008.001.08` JSON schema, so the
output feeds straight into pacs.008 generation.
